Gerenciador de Loja de Flores

Florg

Uma solução de gerenciamento para floriculturas.

Visão geral

Esse projeto visa a construção de um software capaz de fazer o gerenciamento das vendas de uma floricultura. O sistema deve manter o registro dos estoques, dos pedidos e dos clientes da loja. Para os estoques, o sistema deve manter o inventário de todas as flores disponíveis para as vendas, controlar quantas flores de cada tipo chegaram e a data de chegada. Cada cliente pode ser cadastrado com informações de contato e servem exclusivamente para o controle de pedidos. Os pedidos devem manter, para cada cliente, a quantidade e preço de venda unitária para cada produto comprado, além da data de entrega do pedido. Pedidos com data futura são enxergados como encomendas. Com essas informações, o sistema deve ser capaz de fazer dois tipos de relatórios: relatórios de receita e relatórios de demanda. Os relatórios de receita apresentam a receita mensal da loja, seperada por produto e ou cliente. Os relatórios de demanda apresentam o estoque atual e futuro, a quantidade disponível para compra em um determinado dia e as encomendas futuras.

Valor

Somos um negócio para negócios (B2B) e nosso maior valor é organizacional. O software permite armazenar e mostrar as informações pertinentes a pedidos e clientes, de modo que o gerente possa organizar todos os dados pertinentes ao fluxo de suas mercadorias.
